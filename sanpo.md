1. プロジェクト概要日常の「歩き」をモンスターの成長と物語に変換する、1体限定の育成×位置情報散歩アプリ。
2. MAP基盤設計（フェーズ1）2.1. データ取得・技術スタック地図ソース: OpenStreetMap (OSM) ※Overpass APIを使用描画エンジン: MapLibre GL JS または Unity/Flutterによる自作レンダラー位置情報: 標準Geolocation API (A-GPS)歩数計測: Google Fit / Apple Health APIとの連携
2.2. マップ描画手順データの抽象化: リアルな地図画像は使わず、道路(Line)とエリア(Polygon)の幾何学データのみを取得。動的レンダリング: 取得したデータをゲームの世界観（ファンタジー風、ドット絵風など）のテクスチャで上書き描画。フォグ・オブ・ウォー: 自分の周囲（半径50m〜100m）以外は霧で隠し、探索感を演出。
2.3. GPS補正ロジックドリフト対策: カルマンフィルタによる位置の平滑化。歩数優先同期: GPSが移動しても歩数計が動いていない場合は、キャラクターを「足踏み」状態にする。
3. 環境依存型成長システム
3.1. エリア判定ロジック (OSMタグ連動)OSMのタグ情報を参照し、歩行中の環境をリアルタイム判定する。カテゴリOSMタグ判定基準成長影響ステータス都会highway, shop, building密度高器用さ・賢さ自然leisure=park, natural=wood体力・回復力水辺natural=water, waterway素早さ・回避田舎建物タグが極端に少ないエリア攻撃力・根性
3.2. 変異要素時間帯: 昼（光/炎適正） / 夜（闇/氷適正）歩行ペース: ゆっくり（防御/知能） / 早歩き（攻撃/素早さ）
4. ゲームサイクルと継続の鍵唯一無二の相棒: 仲間を増やさず1体を育てることで、ユーザーの生活の写し鏡とする。足跡マップ: 歩いたルートが「浄化」されたり、色が塗られたりする達成感。非同期交流: マップ上に他プレイヤーの「残響（影）」がたまに通り過ぎる程度のゆるい繋がり。プライバシー保護: 自宅・職場周辺に「隠れ家」を設定し、位置情報を秘匿する機能。
5. 開発ロードマップ（概算：7ヶ月〜）1-3ヶ月目：MAP基盤構築現在地周辺のOSMデータ取得テスト。「今どこにいるか（都会/公園）」の判定エンジン完成。簡易的な自作マップ描画。4-5ヶ月目：育成ロジック実装歩数×環境フラグによるステータス上昇計算。分岐進化システムの構築。6ヶ月目〜：演出・ストーリー追加累計歩数に応じたミニストーリーイベントの実装。UI/UXのブラッシュアップとデバッグ。
